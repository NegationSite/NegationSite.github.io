使用hexo搭建一个博客

Posted on 2022-12-16

本文是使用GitHub Page + hexo 在WIN中来搭建一个属于自己的个人博客。如果遇上下载链接打不开或者下载缓慢，以及安装失败的情况，那就考虑使用魔法上网。

软件安装

本部分需要安装hexo，官方也提供了一个使用文档，安装 Hexo 相当简单，需要先安装下列应用程序即可：

  1. Node.js (Node.js 版本需不低于 10.13，建议使用 Node.js 12.0 及以上版本)
  2. Git

所以如果有安装这两个软件可以直接安装hexo。

这两个软件的具体安装也可以参照hexo的使用文档进行

git 的安装

在安装git之前，我们可以提前注册一个Github账户。

假如你之前从没有听说过git，那么你需要了解一下什么是Git，再考虑怎么安装Git。在这里推荐廖雪峰的Git教程，想要完整的了解和使用可以看看这个教程。我在此简单的概括一下：git是一个分布式版本控制系统。我们现在所搭建的博客最开始都保存在本地，需要上传到GitHub上，而git就是上传的工具。

我们可以在git官网下载并安装，但是我更推荐下载git的镜像文件安装包，这样下载更快一些。一路使用默认设置即可, 我也是新手搞不懂里面有啥, 一般情况不会出问题。

在文件夹页面点击鼠标右键，出现Git Bash Here 和Git GUI Here 就说明安装成功，之后则需要对git进行一番配置：

打开Git Bash Here，输入如下命令：

git config –global user.name “你的GitHub用户名” # 在””中输入,下同
git config –global user.email “你的GitHub注册邮箱”

例如：

git config –global user.name “NegationSite”

想要查看配置信息可以输入命令：

git config -l #小写字母L

或者

git config –list

然后输入命令生成ssh key:

ssh-keygen -t rsa -C “你的GitHub注册邮箱”

按下三次回车后在C:\Users\Username.ssh中生成id_rsa.pub 文件，打开文件并全部复制。打开Github,在sitting页面中选择SSH and GPG keys，选择建立一个新的ssh key，在key 中粘贴你的ssh key。然后点击Add ssh key即可

我们可以在Git Bash Here中输入测试是否成功：

ssh git@github.com

Node.js 的安装()

安装 Node.js 需要请确保勾选 Add to PATH 选项（默认已勾选），也会同时安装及npm。安装成功后可以输入命令查看其安装版本（version）检测是否安装成功：

node -v
npm -v

hexo的安装

所有必备的应用程序安装完成后，即可使用 npm 安装 Hexo。

npm install -g hexo-cli

安装 Hexo 完成后，在你准备保存博客文件的位置上执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件：

hexo init Blog #”Blog”可以换成其它你想设置的名字
cd Bolg # 进入文件目录
npm install

安装Git部署插件，输入命令：

npm install hexo-deployer-git –save

新建完成后，指定文件夹的目录如下：

.
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

每个文件中有什么，代表什么，具体的可以查看hexo的使用文档，在此只介绍几个重要的：

  1. _config.yml
    网站的配置信息，您可以在此配置大部分的参数。其为站点配置文件
  2. themes
    主题文件夹。Hexo会根据主题来生成静态页面。在这个里面也有一个_config.yml，但是这个是用来配置主题的相关信息，其为主题配置文件
  3. /source/_posts
    这个目录里面存放我们所书写的博客
    在所生成的文件夹中输入命令：

    hexo g
    hexo s

打开 http://localhost:4000 ，就可以看到你将要布置的博客大概是一个什么样子。

存在一个打不开的情况。我没有遇到过，但是通过搜索可以发现有人出现这种情况并提供了一个解决方式。

在GitHub中创建一个username.github.io的仓库,例如：NegationSite.github.io

然后在站点配置文件_config.yml中找到deploy修改配置：

deploy:
type: git
repo: https://github.com/username/username.github.io.git #之前在GitHub创建仓库的完整路径+ .git
branch: master

例如：

deploy:
type: git
repo: https://github.com/NegationSite/NegationSite.github.io.git
branch: master

然后执行下列命令
最后执行以下命令：

hexo clean # 清除缓存
hexo g # 生成静态网页
hexo d # 部署到Github

如果成功的话，打开

username.github.io

例如：

NegationSite.github.io

就可以看到刚刚所部署的博客了！（可能需要等待一会）
